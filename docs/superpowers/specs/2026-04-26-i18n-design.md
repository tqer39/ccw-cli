# i18n Support for ccw-cli

Status: Draft (2026-04-26)

## 背景と目的

ccw-cli の出力（picker TUI、tips、`--help`、CLI メッセージ）は現状英語のみ。
日本語ユーザー向けに、システム locale を検出して TUI 出力を切り替えられるようにする。

- 対応言語: English / 日本語の 2 言語のみ
- デフォルト: English
- 将来 OSS 拡大時に他のコントリビューターが翻訳を追加・修正しやすい構造にする

## 翻訳対象スコープ

含める:

- picker の TUI 全般 — メイン一覧、アクションメニュー、削除確認、bulk 削除画面、prune 確認
- `internal/tips/tips.go` の tip 文字列
- `--help` のヘルプ全文（`internal/cli/help.go`）
- CLI のエラー / 情報メッセージ（`internal/picker/run.go` などの `fmt.Fprintln` 系）

含めない:

- ステータスラベル: `PUSHED` / `LOCAL` / `DIRTY` / `PRUNE` / `OPEN` / `DRAFT` / `MERGED` / `CLOSED`
  （GitHub 用語かつスクリプトでも grep されうる識別子的役割。英語固定）
- 色制御コード、`NO_COLOR` 等の環境変数名
- ログ／デバッグ出力（`CCW_DEBUG=1` 時の出力は英語固定）

## 言語決定ロジック

優先順位:

1. CLI フラグ `--lang=en|ja`（最優先）
   - 無効値（例: `--lang=fr`）はエラーで `exit 1`
2. 環境変数 `CCW_LANG=en|ja`
   - 無効値は警告なしで次のソースへフォールスルー
3. システム locale: `LC_ALL` → `LC_MESSAGES` → `LANG` の順で最初に値を持つものを使用
   - 値の正規化: `ja_JP.UTF-8@modifier` → `ja_JP.UTF-8` → `ja_JP` → `ja`
   - primary tag が `ja` なら Japanese、それ以外は English
4. フォールバック: English

`Init()` は `main()` でフラグパース直後に 1 度だけ呼び、以降は `i18n.T()` がシングルトンを参照する。

## アーキテクチャ

新規パッケージ `internal/i18n/`:

```text
internal/i18n/
├── i18n.go          # 公開 API: Init / T / Current / Lang / Key 定数
├── detect.go        # システム locale 検出ロジック
├── detect_test.go
├── i18n_test.go     # キー網羅・%s 数の整合チェック
└── locales/
    ├── en.yaml
    └── ja.yaml
```

YAML は `go:embed` で埋め込み、ビルド成果物に同梱（外部ファイル不要）。

### 公開 API

```go
package i18n

type Lang string

const (
    LangEN Lang = "en"
    LangJA Lang = "ja"
)

type Key string

const (
    KeyTipRename            Key = "tip.rename"
    KeyTipFromPR            Key = "tip.fromPR"
    KeyTipCleanAll          Key = "tip.cleanAll"
    KeyTipPassthrough       Key = "tip.passthrough"
    KeyTipResumeBadge       Key = "tip.resumeBadge"

    KeyPickerHeader         Key = "picker.header"
    KeyPickerActionMenu     Key = "picker.action.menu"
    KeyPickerDeleteConfirm  Key = "picker.delete.confirm"
    KeyPickerPruneConfirm   Key = "picker.prune.confirm"
    KeyPickerPruneBulk      Key = "picker.prune.bulk"
    KeyPickerBulkSelect     Key = "picker.bulk.select"
    KeyPickerBulkConfirm    Key = "picker.bulk.confirm"
    // ... picker 内に存在する全 user-facing 文字列分

    KeyHelpUsage            Key = "help.usage"

    // CLI 系エラー / 情報
    KeyErrInvalidLang       Key = "err.invalidLang"
    // ... 必要に応じて追加
)

// Init resolves the active language and loads message catalogs.
// flagLang は --lang の値（空文字列なら env / system locale にフォールバック）。
// 無効な flagLang は error を返す。
func Init(flagLang string) error

// T returns the translated string for key, formatted with fmt.Sprintf semantics.
// 未定義のキーが渡された場合は key 名そのものを返す（fail-soft）。
func T(key Key, args ...any) string

// Current returns the active language. テスト/デバッグ用。
func Current() Lang
```

### YAML スキーマ

階層キーを `.` で flatten し、Go 側 `Key` 定数と一対一対応させる。
複数行は YAML ブロックスカラー（`|`）で記述。

```yaml
# locales/en.yaml
tip:
  rename: "Worktree name = session name; renaming with /rename is fine, ccw doesn't track it"
  fromPR: "claude --from-pr <number> resumes a PR-linked session directly"
  cleanAll: "--clean-all sweeps pushed worktrees in bulk"
  passthrough: "ccw -- --model <id> passes flags through to claude"
  resumeBadge: "The RESUME badge is derived from ~/.claude/projects/"

picker:
  header: "Select a worktree:"
  action:
    menu: |
      Selected: %s (%s)
      Path:     %s

      What to do?
        [r] run    — start claude in this worktree
        [d] delete — remove the worktree
        [b] back   — return to list
        [q] quit   — cancel
  delete:
    confirm: |
      Delete worktree %s?
        path:   %s
        status: %s

      This will run: %s

      Confirm? [y/N]
  # ... 残りのキー

help:
  usage: |
    Usage: ccw [options] [-- <claude-args>...]
    ...

err:
  invalidLang: "invalid --lang value: %s (expected 'en' or 'ja')"
```

`ja.yaml` は同一キーセットで日本語訳を持つ。

## 移行対象ファイル

| ファイル | 変更内容 |
|---|---|
| `internal/tips/tips.go` | `defaults` を `[]i18n.Key` に変更し、`Defaults()` で `i18n.T()` で都度解決 |
| `internal/cli/help.go` | `usage` 定数を削除し、`PrintHelp` 内で `i18n.T(i18n.KeyHelpUsage)` を呼ぶ |
| `internal/cli/parse.go` | `--lang` フラグを追加（値検証含む） |
| `internal/picker/view.go` | 全 user-facing literal を `i18n.T(...)` 呼び出しに置換 |
| `internal/picker/run.go` | `"Select a worktree:"` 等を `i18n.T(...)` に置換 |
| `internal/picker/bulk.go` | bulk 削除 UI 文言を `i18n.T(...)` に置換 |
| `cmd/ccw/main.go` | フラグパース直後に `i18n.Init(opts.Lang)` を呼ぶ |
| ステータスラベル / 色制御 | 変更なし（英語固定） |

## テスト戦略

`internal/i18n/i18n_test.go`:

- 両 YAML が `embed.FS` から正しくパースされる
- Go 定数の `Key` 集合 = `en.yaml` のキー集合 = `ja.yaml` のキー集合
  （差分があればテスト失敗、新規キー追加忘れを CI で必ず検出）
- 各キーで `%s` / `%d` / `%v` 等のフォーマット verb の種類と並びが en/ja で一致

`internal/i18n/detect_test.go`:

- フラグ × 環境変数 × system locale のマトリクスで期待 `Lang` を返すことを確認
- `LC_ALL` > `LC_MESSAGES` > `LANG` の優先順位
- `ja_JP.UTF-8@modifier` 形式の正規化
- 無効な `--lang` 値はエラー、無効な `CCW_LANG` 値は次のソースへフォールスルー

既存テスト（`tips`, `picker`, `cli`）への影響:

- 各テストパッケージの `TestMain` で `i18n.Init("en")` を呼び、出力を English で安定化
- スナップショット的な期待文字列を持つテストはそのまま動く

## tape / docs / CI への影響

- `tests/vhs/*.tape`: 先頭に `Env CCW_LANG en` を追加し、デモ GIF を English 固定
- `README.md`: `--lang` / `CCW_LANG` の節を Environment セクションに追加
- `docs/README.ja.md`: 同等の節を日本語で追加
- `lefthook.yml` / GitHub Actions: 既存の `go test ./...` で parity チェックが回るので追加変更なし

## YAML パーサ依存

`gopkg.in/yaml.v3` を `go.mod` に追加（広く使われており、Go コミュニティの事実上の標準）。

## 非目標

- 言語の動的切り替え（実行中に言語を変える機能）
- 複数形ルール（CLDR）対応 — 現状の文面に必要な箇所がない
- locale 別の数値 / 日付フォーマット — 該当出力なし
- 翻訳ファイルの外部上書き（`~/.config/ccw/locales/` 等） — YAGNI
- en / ja 以外の言語 — 必要になった時点で追加
